Clinical Instruction API

End Points

There is only one endpoint available for this API and this can be accessed via the, /instruct endpoint. The endpoint takes the parameter text which instructs the underlying fine-tuned GPT2 LLM the instruction to carry out. Further explanation is given below.

/instruct POST

Parameter Type Position # Description
text string body Required Text to query the fine-tuned GPT2 model.
max_num_tokens int body Optional The maximum number of tokens to be generated by the model. Defaults to 35 if no parameter is provided
temperature int body Optional Controls the randomness and creativity of the model's output.
Lower value(e.g 0.2) makes the model more deterministic.
Higher value(e.g 0.8 and above) introduces more randomness in the model's output.
Temperature default(1) represents the model's inherent probability distribution
top_k int body Optional Limits the model's choices when generating text to the k most likely next tokens.
Default: 5.

Response

{
    "status": true,
    "result_code": 200,
    "message": "Success!",
    "values": {
        "Response": "model generated text",
    }
}
